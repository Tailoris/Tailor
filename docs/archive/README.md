# Tailor IS 文档归档区

## 归档规则

### 归档原则
1. **按月归档**：文档按原始创建/修改月份归类，归档时记录归档时间。
2. **记录原因**：每份归档文档需说明归档原因（如：版本已迭代、任务已完成、报告已过期）。
3. **保留索引**：通过 `archive-index.md` 维护完整的归档文档索引，支持快速查找。

### 归档条件
满足以下任一条件的文档应归档：
- 阶段性报告已完成后续迭代（如 Sprint 完成报告、审计报告）
- 旧版本部署方案/清单已被新版替代
- 修复清单/计划已完成执行
- 核查工作指令/方案已执行完毕
- 临时性文档（如清理清单、风险评估）

### 保留规则
以下文档保留在 `docs/reports/` 中：
- 当前活跃的 Issue 追踪
- 当前季度的核查验证清单
- 当前使用的代码质量改进方案
- 当前业务任务追踪
- 当前值班排班表
