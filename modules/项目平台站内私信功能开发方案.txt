# Tailor IS（裁智云）项目平台站内私信功能开发指令

### 一、功能概述
在TailorIS项目中实现用户间站内私信通信功能，支持文本、图片、文件及站内链接发送，并具备敏感内容过滤机制。

### 二、业务需求详述

#### 1. 核心功能
- **私信收发**：用户间可互发私信（支持单聊）
- **内容类型**：
  - 文本消息（支持富文本格式）
  - 图片（支持常见格式：JPG、PNG、WebP等）
  - 文件（支持DXF、PDF、PRJ等业务文件格式）
  - 站内链接（自动识别并转换为可点击链接）
- **消息管理**：
  - 对话列表（按最近消息排序）
  - 消息历史记录（支持分页加载）
  - 消息状态（已发送、已送达、已读）
  - 消息撤回（2分钟内可撤回）
  - 消息删除（软删除，仅对己方不可见）

#### 2. 安全限制
- **禁止内容**：
  - 联系方式：手机号、微信号、QQ号、邮箱、社交账号等
  - 敏感词汇：包含政治敏感、色情、暴力、广告营销等关键词
  - 外部链接：仅允许站内链接（如商品、用户、帖子链接）
- **过滤机制**：
  - 发送前实时检测
  - 违规内容自动拦截并提示
  - 支持人工审核队列（可疑消息转入人工审核）

#### 3. 用户体验
- **实时通信**：WebSocket实时推送新消息
- **离线消息**：支持离线消息存储与推送
- **文件管理**：
  - 图片预览（支持缩放）
  - 文件大小限制：图片≤3MB，文件≤5MB
  - 文件类型白名单校验
- **通知提醒**：
  - 桌面通知（用户授权后）
  - 未读消息计数（全局显示）
  - 声音提示（可配置开关）

### 三、技术要求

#### 1. 技术栈
- **前端**：
  - Vue 3 + Composition API
  - Element Plus组件库
  - WebSocket (Socket.io或原生WebSocket)
  - File API + Canvas（图片压缩）
- **后端**：
  - Spring Boot 3.x
  - WebSocket (Spring WebSocket或Socket.IO Java)
  - MyBatis Plus
  - MinIO（文件存储）
- **安全过滤**：
  - AC自动机算法（敏感词匹配）
  - 正则表达式（联系方式识别）
  - Redis缓存（敏感词缓存）

#### 2. 微服务设计
- **服务归属**：在现有`message-service`中扩展实现
- **新增模块**：
message-service
├── private-chat-controller # 私信REST接口
├── private-chat-service # 私信业务逻辑
├── websocket-handler # WebSocket处理
├── content-filter # 内容过滤组件
└── file-upload-handler # 私信文件上传


### 四、具体开发任务

#### 1. 数据库设计
**指令**：设计站内私信相关数据库表结构

**需要设计的表**：
1. `private_chat_session`（私信会话表）
2. `private_chat_message`（私信消息表）
3. `sensitive_keyword`（敏感词库表）
4. `chat_message_audit`（消息审核记录表）

**要求**：
- 合理的字段设计（包含会话ID、发送者、接收者、消息类型、内容、状态等）
- 索引优化（按会话ID、用户ID、创建时间建立索引）
- 外键约束
- 软删除支持（is_deleted字段）
- 审计字段（created_time, updated_time）

**请输出**：
- 完整的DDL建表语句（MySQL 8.0）
- 索引设计说明
- 表关系图（ER图）

---

#### 2. 后端接口开发
**指令**：在`message-service`中实现私信REST API

**接口清单**：
1. `POST /api/private-chat/sessions` - 创建私信会话
2. `GET /api/private-chat/sessions` - 获取会话列表
3. `GET /api/private-chat/sessions/{sessionId}/messages` - 获取消息历史
4. `POST /api/private-chat/messages` - 发送私信
5. `PUT /api/private-chat/messages/{messageId}/recall` - 撤回消息
6. `PUT /api/private-chat/messages/{messageId}/read` - 标记已读
7. `POST /api/private-chat/upload` - 上传私信文件
8. `GET /api/private-chat/unread-count` - 获取未读消息数

**技术要求**：
- Spring Boot 3.x + MyBatis Plus
- 统一响应格式（BaseResponse）
- 全局异常处理
- 参数校验（@Validated）
- 接口级权限控制（只能操作自己的私信）

**请输出**：
- PrivateChatController.java（完整代码）
- PrivateChatService.java（业务逻辑）
- PrivateChatMessage.java（实体类）
- PrivateChatMessageMapper.java（数据访问层）
- 请求/响应DTO类定义

---

#### 3. WebSocket实时通信
**指令**：实现WebSocket私信实时推送功能

**功能要求**：
1. 建立WebSocket连接（带Token认证）
2. 实时消息推送（新消息、消息已读、消息撤回等事件）
3. 连接状态管理（在线、离线、重连）
4. 消息确认机制（确保消息送达）

**技术实现**：
- Spring WebSocket + STOMP协议
- 或 Socket.IO Java实现
- Redis存储在线状态
- 消息持久化（保证离线消息不丢失）

**请输出**：
- WebSocketConfig配置类
- ChatWebSocketHandler处理类
- 消息事件定义（MessageEvent.java）
- 前端WebSocket连接工具类

---

#### 4. 内容安全过滤
**指令**：实现私信内容安全过滤组件

**过滤规则**：
1. **联系方式检测**（使用正则表达式）：
 - 手机号（中国大陆+国际格式）
 - 邮箱地址
 - 微信号/QQ号模式
 - 社交账号特征
2. **敏感词过滤**：
 - AC自动机算法匹配
 - 支持敏感词库动态更新
 - 支持模糊匹配（拼音、形近字）
3. **链接检测**：
 - 只允许站内链接（域名白名单）
 - 外部链接自动拦截

**功能要求**：
- 发送前实时检测
- 违规内容拦截并返回具体原因
- 可疑内容转入人工审核队列
- 支持学习模式（收集新敏感词）

**请输出**：
- ContentFilterService.java（过滤服务）
- SensitiveWordMatcher.java（敏感词匹配器）
- ContactDetector.java（联系方式检测器）
- LinkValidator.java（链接验证器）
- 敏感词库初始化SQL

### 五、集成与部署要求

#### 1. 与现有系统集成
- 用户身份验证（复用现有JWT机制）
- 文件存储（复用MinIO服务，但私信文件单独存储桶）

#### 2. 配置文件示例
```yaml
# application-chat.yml
chat:
websocket:
  enabled: true
  endpoint: /ws/chat
  allowed-origins: "*"

security:
  filter-enabled: true
  allow-external-links: false
  max-file-size: 5MB
  image-formats: jpg,jpeg,png,gif,webp
  
message:
  recall-timeout: 60 # 撤回时间限制（秒）
  history-keep-days: 30 # 历史消息保留天数


## 使用说明

1. **分阶段执行**：可以将上述指令拆分为多个子任务，按优先级分阶段开发
2. **优先级建议**：
   - 第一阶段：数据库设计 + 基础REST API
   - 第二阶段：WebSocket实时通信
   - 第三阶段：内容安全过滤
   - 第四阶段：前端组件开发
   - 第五阶段：管理后台功能

3. **与现有系统集成**：注意复用已有的用户认证、文件服务等组件，避免重复开发

4. **测试重点**：内容过滤准确性和实时通信稳定性是关键测试点
