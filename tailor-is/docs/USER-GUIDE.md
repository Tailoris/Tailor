## 1. 平台概述

### 1.1 平台介绍

**Tailor IS（裁智云）**是一个面向服装全产业链的数字化定制设计交易平台。平台覆盖从服装设计、AI 辅助制版、版权登记保护，到商品上架、交易、营销、售后的完整业务链路，同时提供社区互动、在线教育、数据分析等增值服务。

平台采用 **微服务架构**，基于 Spring Cloud 2023 + Spring Boot 3.2 生态构建，包含 **16 个独立微服务**，通过 API Gateway 统一入口对外提供服务。

### 1.2 平台架构

| 层级 | 说明 |
|------|------|
| 客户端层 | PC 管理端、商户端、H5 移动端、小程序 / App |
| 网关层 | Spring Cloud Gateway — 路由转发、认证鉴权、限流、API 文档聚合 |
| 业务服务层 | 16 个微服务，覆盖用户、商户、商品、订单、支付、营销、AI、版权、社区、供应链、管理、消息、版型、即时通讯、学院、数据分析等完整业务 |
| 基础设施层 | MySQL、Redis、RabbitMQ、Nacos、Nginx、MinIO、ELK、Prometheus、Grafana |

### 1.3 微服务一览

| 序号 | 服务名称 | 端口 | 职责描述 |
|------|----------|------|----------|
| 1 | **gateway** | 8080 | API 网关 — 统一入口、路由分发、鉴权过滤 |
| 2 | **admin** | 8100 | 后台管理 — 平台运营、数据看板、审核仲裁 |
| 3 | **user** | 8101 | 用户服务 — 注册登录、权限管理、收货地址、实名认证 |
| 4 | **product** | 8102 | 商品服务 — 商品发布与管理、SKU、分类、评价 |
| 5 | **order** | 8103 | 订单服务 — 订单创建、状态流转、售后工单、购物车 |
| 6 | **payment** | 8104 | 支付服务 — 微信支付、支付宝、退款、对账、结算 |
| 7 | **marketing** | 8105 | 营销服务 — 优惠券、秒杀、拼团、会员体系、积分 |
| 8 | **ai** | 8106 | AI 服务 — 智能量体、图案生成、图案检查 |
| 9 | **copyright** | 8107 | 版权服务 — 版权登记、区块链存证、侵权仲裁 |
| 10 | **community** | 8108 | 社区服务 — 帖子、评论、点赞、关注、举报 |
| 11 | **supply** | 8109 | 供应链服务 — 供应商管理、采购、物流协同 |
| 12 | **merchant** | 8110 | 商户服务 — 商户入驻审核、店铺管理、员工管理 |
| 13 | **message** | 8111 | 消息服务 — 系统通知、模板消息、消息推送 |
| 14 | **academy** | 8112 | 学院服务 — 课程管理、章节管理、学习记录 |
| 15 | **analytics** | 8113 | 数据分析 — 业务指标、数据看板、报表 |
| 16 | **message-im** | 8114 | 即时通讯 — 实时聊天、会话管理 |

### 1.4 前端入口

## 2. 快速开始

### 2.1 环境要求

| 组件 | 最低版本 | 推荐版本 | 用途 |
|------|----------|----------|------|
| JDK | 17 | 17 LTS | Java 运行环境 |
| Maven | 3.8 | 3.9+ | 项目构建 |
| MySQL | 8.0 | 8.0.35+ | 关系型数据库 |
| Redis | 7.0 | 7.2+ | 缓存与 Session 存储 |
| RabbitMQ | 3.12 | 3.13+ | 消息队列（需启用 delayed_message_exchange 插件） |
| Nacos | 2.x | 2.3+ | 注册中心 & 配置中心 |
| Docker | 24.0 | 24.0.7+ | 容器化部署 |
| Node.js | 18 | 20 LTS | 前端构建 |

### 2.2 安装部署

#### 方式一：Docker Compose 一键部署（推荐）

```bash
# 1. 克隆项目
git clone <repository-url>
cd tailor-is

# 2. 启动全部服务（含基础设施 + 微服务）
docker compose up -d

# 3. 检查服务状态
docker compose ps

# 4. 初始化数据库
bash deploy/scripts/init-database.sh

# 5. 验证部署
curl http://localhost:8080/actuator/health
```

#### 方式二：手动部署

**步骤 1 — 启动基础设施**

```bash
docker compose -f docker-compose.yml up -d mysql redis rabbitmq nacos
```

**步骤 2 — 数据库初始化**

```bash
# 依次导入 SQL 脚本
for f in sql/*.sql; do
    docker exec -i tailor-mysql mysql -uroot -p${MYSQL_ROOT_PASSWORD} < "$f"
done
```

**步骤 3 — 编译打包**

```bash
mvn clean install -DskipTests
```

**步骤 4 — 按依赖顺序启动微服务**

```
第一阶段: gateway → admin → user
第二阶段: merchant → product → order → payment
第三阶段: marketing → ai → copyright → community → supply → message → pattern → message-im → academy → analytics
```

或使用一键启动脚本：

```bash
bash deploy/scripts/start-all-services.sh
```

**步骤 5 — 前端部署**

```bash
# PC 商城
cd tailor-is-frontend/pc-mall && npm install && npm run build

# 商户端
cd tailor-is-frontend/merchant-admin && npm install && npm run build

# 平台管理后台
cd tailor-is-frontend/platform-admin && npm install && npm run build

# 移动端（UniApp）
cd tailor-is-frontend/mobile-app && npm install
```

### 2.3 首次启动验证

| 检查项 | 验证方法 |
|--------|----------|
| Nacos 就绪 | 浏览器访问 `http://localhost:8848/nacos`，查看所有服务已注册 |
| Gateway 正常 | `curl http://localhost:8080/actuator/health` 返回 `{"status":"UP"}` |
| API 文档 | 浏览器访问 `http://localhost:8080/doc.html` |
| 前端页面 | 将 Nginx 指向前端 dist 目录，访问对应地址 |

## 3. 用户功能指南

### 3.1 注册与登录

#### 3.1.1 注册账号

1. 访问平台首页，点击 **「注册」**
2. 输入手机号，获取并填写短信验证码
3. 设置登录密码（8–20 位，需包含字母 + 数字）
4. 阅读并同意《用户服务协议》和《隐私政策》
5. 点击 **「注册」** 完成账号创建

> 注册成功后系统自动为用户创建钱包账户（位于 `tailor_payment.user_account`）。

#### 3.1.2 登录

支持以下登录方式：

| 方式 | 说明 |
|------|------|
| 手机号 + 密码 | 常规登录 |
| 手机号 + 验证码 | 免密登录 |
| 邮箱 + 密码 | 已绑定邮箱的用户 |

> 安全机制：连续 5 次登录失败将锁定账号 30 分钟；系统自动记录登录设备信息。

### 3.2 浏览商品

#### 3.2.1 首页浏览

- **轮播 Banner**：平台活动、热门商品推荐
- **分类导航**：按品类（男装、女装、童装、面料辅料等）快速浏览
- **推荐专区**：基于浏览/购买历史的个性化推荐

#### 3.2.2 搜索与筛选

- 关键词搜索商品标题、描述、品牌
- 多维度筛选：价格区间、销量、评价、品牌、尺码、颜色
- 排序方式：综合、销量、价格、好评、上新时间

#### 3.2.3 商品详情

- 商品主图轮播（最多 8 张），支持视频展示
- SKU 选择：颜色、尺码等规格组合
- 价格信息：销售价、市场价格（划线价）、会员价
- 商品参数：材质、版型、厚度、弹性等属性
- 用户评价：文字 + 图片评价，支持筛选
- 客服咨询：点击 **「联系客服」** 进入 IM 聊天

### 3.3 下单购买

#### 3.3.1 购物车

- 添加商品到购物车，选择规格和数量
- 支持批量勾选、全选操作
- 实时显示价格小计
- 购物车数据本地缓存 + 云端同步

#### 3.3.2 确认订单

1. 选择收货地址（可新增/修改/设为默认）
2. 选择配送方式
3. 选择优惠券（系统自动计算最优优惠）
4. 使用积分抵扣（如有）
5. 备注留言（选填）
6. 确认商品清单和最终金额

#### 3.3.3 提交订单

- 提交后订单状态为 **「待付款」**
- 系统自动锁定库存
- **30 分钟内未完成支付**，订单自动取消并释放库存

### 3.4 支付流程

支持以下支付方式：

| 支付方式 | 说明 |
|----------|------|
| **微信支付** | 扫码支付 / JSAPI 支付 / APP 支付 |
| **支付宝** | 扫码支付 / 手机网站支付 / APP 支付 |

**支付流程**：

1. 在订单确认页点击 **「去支付」**
2. 选择支付方式
3. 跳转至支付页面完成支付
4. 支付成功后自动跳转至订单详情页
5. 系统自动发放积分、更新会员等级（通过 RabbitMQ 异步处理）

> 支付安全保障：所有支付接口经过签名校验，支付回调通过验签机制确保数据未被篡改。

### 3.5 订单管理

#### 3.5.1 订单状态流转

```
待付款 → 待发货 → 待收货 → 已完成
   ↓        ↓
已取消   申请售后 → 售后处理中 → 售后完成
```

#### 3.5.2 操作说明

| 操作 | 可用状态 | 说明 |
|------|----------|------|
| **取消订单** | 待付款 | 用户主动取消，自动释放库存 |
| **催促发货** | 待发货 | 向商户发送催发货通知 |
| **确认收货** | 待收货 | 用户确认已收到商品 |
| **申请售后** | 待收货 / 已完成 | 发起退款 / 退货退款申请 |
| **查看物流** | 待发货 / 待收货 | 跟踪快递物流信息 |

> 自动确认：发货后 **15 天** 用户未手动确认，系统自动确认收货。

### 3.6 个人中心

#### 3.6.1 账户设置

- **修改密码**：需验证原密码或手机验证码
- **绑定手机号**：更换绑定手机
- **绑定邮箱**：设置备用联系方式
- **实名认证**：上传身份证信息（部分功能需实名认证）

#### 3.6.2 地址管理

- 新增 / 编辑 / 删除收货地址
- 设置默认地址
- 支持批量导入地址

#### 3.6.3 我的收藏

- 收藏商品：浏览商品时点击收藏
- 收藏店铺：关注感兴趣的商户
- 收藏管理：批量取消收藏

#### 3.6.4 优惠券与积分

| 功能 | 说明 |
|------|------|
| **我的优惠券** | 查看可用 / 已使用 / 已过期的优惠券 |
| **我的积分** | 查看积分余额、积分获取 / 消耗记录 |
| **积分兑换** | 用积分兑换优惠券、礼品等 |

#### 3.6.5 我的订单

- 全部订单 / 待付款 / 待发货 / 待收货 / 已完成 / 售后
- 支持按时间范围筛选
- 支持搜索订单编号、商品名称

#### 3.6.6 钱包与资产

| 项目 | 说明 |
|------|------|
| **余额** | 平台账户余额，可用于支付 |
| **充值** | 通过微信 / 支付宝充值 |
| **提现** | 提现至银行卡或支付宝 |

## 4. 商户功能指南

### 4.1 商户入驻

#### 4.1.1 注册账号

1. 访问平台首页，点击 **「商户入驻」**
2. 填写手机号 + 短信验证码
3. 设置登录密码（8–20 位，字母 + 数字）
4. 完成注册

#### 4.1.2 提交商户资料

1. 登录后进入 **「商户中心」→「入驻申请」**
2. 选择商户类型：

| 类型 | 适用对象 | 资质要求 |
|------|----------|----------|
| **个人商户** | 个人设计师、版师 | 身份证正反面 |
| **企业商户** | 服装公司、工厂、品牌商 | 营业执照 + 法人身份证 |

3. 填写商户信息：名称、联系人、电话、经营范围
4. 上传资质文件（jpg/png/pdf，单个不超过 5MB）
5. 提交审核

> 审核周期：1–3 个工作日。审核结果通过站内消息 / 短信通知。

#### 4.1.3 店铺初始化

审核通过后，完成以下设置：

1. **店铺设置**：
   - 店铺名称（提交后不可随意修改）
   - 店铺 Logo（建议 200×200px）
   - 店铺 Banner（建议 1200×400px）
   - 店铺简介（200 字以内）
   - 客服联系方式
2. **运费模板**：设置不同地区的运费规则
3. **退货地址**：设置售后退货地址
4. **缴纳质保金**：根据商户类型缴纳对应金额质保金，支持微信 / 支付宝 / 银行转账

### 4.2 商品管理

#### 4.2.1 发布商品

1. 进入 **「商品管理」→「商品列表」→「发布商品」**
2. 填写商品基本信息：

| 字段 | 要求 |
|------|------|
| 商品标题 | ≤60 字，建议：品牌 + 款式 + 材质 + 场景 |
| 商品分类 | 选择至三级分类（如：女装 > 连衣裙 > 长袖连衣裙） |
| 商品描述 | 支持富文本编辑，可插入图片 |
| 商品主图 | 至少 1 张，最多 8 张，建议 800×800px 以上 |
| 商品视频 | 可选，16:9 比例，≤60 秒 |

3. 设置 SKU：颜色、尺码、自定义规格等，为每个 SKU 设置价格、库存、SKU 编码
4. 提交审核

#### 4.2.2 商品状态

| 状态 | 说明 |
|------|------|
| 草稿 | 保存但未提交 |
| 审核中 | 已提交，等待平台审核 |
| 已上架 | 审核通过，正常销售 |
| 已下架 | 商家主动下架或平台强制下架 |
| 审核驳回 | 需修改后重新提交 |

> 注意：仅草稿和已下架状态可编辑商品。已上架商品需先下架再编辑。

#### 4.2.3 批量导入

1. 下载 Excel 导入模板
2. 按模板填写商品信息
3. 上传文件，系统自动校验
4. 预览校验结果，确认导入
5. 导入商品默认为 **「草稿」** 状态

### 4.3 订单处理

#### 4.3.1 订单列表

- 支持按状态、时间、订单编号、买家昵称、商品名称筛选
- 支持批量导出订单数据
- 支持添加内部备注和买家标记

#### 4.3.2 发货操作

1. 进入 **「订单管理」→「待发货」**
2. 点击 **「发货」**
3. 填写物流公司 + 快递单号
4. 确认发货，系统自动通知买家

#### 4.3.3 售后处理

1. 查看买家售后申请（退款 / 退货退款）
2. 根据实际情况处理：

| 处理方式 | 说明 |
|----------|------|
| 同意退款 | 直接退款给买家 |
| 同意退货退款 | 等待买家寄回，收到后确认退款 |
| 拒绝 | 填写拒绝原因，买家可再次申请 |

> 货款结算周期：买家确认收货后 T+7 结算至商户账户。

### 4.4 营销工具

#### 4.4.1 优惠券

| 类型 | 说明 |
|------|------|
| 满减券 | 满 X 元减 Y 元 |
| 折扣券 | 打 X 折 |
| 免邮券 | 免运费 |

发放方式：公开领取 / 主动发放 / 活动关联。

#### 4.4.2 会员体系

设置会员等级（普通 → 银卡 → 金卡 → 钻石），配置各等级权益：

- 消费折扣率（如金卡 9 折）
- 专属优惠券
- 包邮权益
- 生日特权

#### 4.4.3 积分体系

- **获取规则**：消费积分、签到积分、评价奖励积分
- **使用规则**：积分兑换优惠券、商品、服务

#### 4.4.4 营销活动

| 活动类型 | 说明 |
|----------|------|
| 限时折扣 | 指定时间段内打折 |
| 满减活动 | 满额减 |
| 拼团活动 | 多人拼团享受优惠价 |
| 秒杀活动 | 超低价限量抢购 |
| 新品首发 | 新品预热活动 |
| 清仓特卖 | 库存清理特价 |

### 4.5 数据看板

#### 4.5.1 经营概览

| 指标 | 说明 |
|------|------|
| 今日销售额 | 当日已支付订单金额 |
| 今日订单数 | 当日订单数量 |
| 访客数 | 店铺访客 UV |
| 浏览量 | 商品详情页 PV |
| 转化率 | 下单用户数 / 访客数 |
| 客单价 | 销售额 / 订单数 |

#### 4.5.2 商品分析

- 商品销量排行
- 商品收藏排行
- 商品评价分析
- SKU 销售分布

#### 4.5.3 用户分析

- 新老客户占比
- 客户地域分布
- 复购率分析
- 客户消费层级

### 4.6 版权服务

商户可对原创设计进行版权保护：

1. **版权登记**：上传设计作品（jpg/png/ai/psd/pdf），选择平台存证或版权局代理登记
2. **版权查询**：按作品名称 / 登记号 / 权利人 / 类型查询
3. **侵权投诉**：发现盗用设计可在线发起投诉，平台 3 个工作日内处理
4. **版权交易**：在线授权或转让版权，双方签署电子协议

> 平台采用区块链存证 + 时间戳技术，可作为著作权权属的初步证据。

## 7. 常见问题

### 通用问题

**Q: 忘记登录密码怎么办？**
A: 在登录页点击「忘记密码」，通过手机验证码或邮箱验证重置密码。

**Q: 支持哪些浏览器？**
A: 推荐使用 Chrome 90+、Firefox 88+、Safari 14+、Edge 90+ 等现代浏览器。IE 浏览器不受支持。

**Q: 平台是否支持多语言？**
A: 是的，平台前端支持中文 / 英文切换（i18n 国际化）。

### 商品相关

**Q: 商品审核不通过怎么办？**
A: 查看审核驳回原因，按要求修改后重新提交。常见驳回原因：图片不清晰、标题违规、分类错误、价格异常等。

**Q: 可以设置预售商品吗？**
A: 可以。在发布商品时选择「预售」模式，设置预售结束时间和发货时间。

**Q: 商品上架后多久可以在搜索结果中看到？**
A: 商品审核通过后即刻上架，搜索索引同步延迟通常在 1–3 分钟内。

### 订单相关

**Q: 买家一直不确认收货怎么办？**
A: 系统会在发货后自动确认收货（默认 15 天），无需担心。

**Q: 如何修改订单价格？**
A: 在待付款状态下，商户可对订单进行改价操作（仅支持降价）。

**Q: 订单支付超时后可以重新支付吗？**
A: 订单超时取消后需重新下单。未超时的待付款订单可随时继续支付。

### 支付相关

**Q: 支付失败怎么办？**
A: 检查网络连接、余额是否充足、银行卡是否限额。如仍无法支付，可尝试更换支付方式或联系平台客服。

**Q: 支持哪些支付方式？**
A: 支持微信支付和支付宝。

### 资金相关（商户）

**Q: 货款何时到账？**
A: 买家确认收货后，货款进入商户账户；结算周期为 T+7（确认后 7 天）。

**Q: 如何申请提现？**
A: 进入「资金管理」→「提现」，输入金额提交申请，审核通过后 1–3 个工作日到账。

**Q: 平台佣金如何计算？**
A: 按商品品类设置不同的抽佣比例，在订单结算时自动扣除。具体比例可在平台管理后台查看。

### 版权相关

**Q: 平台版权存证有法律效力吗？**
A: 平台采用区块链存证 + 时间戳技术，可作为著作权权属的初步证据。如需正式登记，可选择平台代理向版权局申请。

**Q: 发现他人盗用我的设计怎么办？**
A: 立即在版权服务中发起侵权投诉，上传侵权证据，平台将在 3 个工作日内介入处理。

### 安全相关

**Q: 如何保证账户安全？**
A: 建议：定期修改密码、开启登录短信验证、不与他人共享账号、定期检查登录设备记录、发现异常及时修改密码。

**Q: 平台如何保护用户隐私？**
A: 密码采用 BCrypt 加密存储；敏感数据（手机号、身份证、银行卡）在日志和显示时自动脱敏；所有 API 通信使用 HTTPS 加密传输。
