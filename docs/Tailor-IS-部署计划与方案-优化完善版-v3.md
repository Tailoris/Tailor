#### 3.1.2 部署步骤

```bash
# Step 1: 克隆代码并配置环境
git clone git@github.com:Tailoris/Tailor.git /opt/tailor-is
cd /opt/tailor-is

# Step 2: 配置环境变量（必须填写真实值）
cp deploy/.env.example deploy/.env
# 编辑 deploy/.env，填入：
#   - MYSQL_ROOT_PASSWORD（强密码）
#   - REDIS_PASSWORD（强密码）
#   - RABBITMQ_DEFAULT_PASS（强密码）
#   - NACOS_AUTH_TOKEN（强密码）
#   - GRAFANA_PASSWORD（强密码）
#   - GATEWAY_HMAC_SECRET（随机生成）
#   - AES_KEY（随机生成 256-bit）

# Step 3: 初始化数据库
mysql -h 127.0.0.1 -u root -p < deploy/sql/000_init_database.sql
for f in tailor-is/sql/0*_*.sql; do
  mysql -h 127.0.0.1 -u root -p < "$f"
done

# Step 4: 启动基础设施
docker compose up -d mysql redis rabbitmq nacos

# Step 5: 等待基础设施健康检查通过
docker compose ps

# Step 6: 构建并启动全部服务
docker compose up -d --build

# Step 7: 验证部署
./deploy/scripts/health-check.sh
```

## 四、质量保障体系

### 4.1 CI/CD 流水线

```
Git Push → GitHub Actions
  ├── PR Check: 代码规范 + 废弃文件检测 + 安全扫描
  ├── Backend CI: 编译 → 单元测试 → Checkstyle → PMD → JaCoCo → Docker 构建
  ├── Frontend CI: Lint → 类型检查 → 构建 → E2E 测试
  └── Project Cleanup: 每周定期扫描废弃文件

质量门禁:
  ├── Checkstyle: 0 error（阻断合并）
  ├── PMD: 0 高优违规（阻断合并）
  ├── JaCoCo: ≥ 80% 行覆盖率（阻断合并）
  ├── OWASP DC: 0 Critical/High（阻断合并）
  ├── TruffleHog: 0 敏感信息（阻断合并）
  └── ESLint: 0 error（阻断合并）
```

### 4.2 监控告警

| 层级 | 工具 | 监控内容 | 告警阈值 |
|------|------|---------|---------|
| 基础设施 | Prometheus + Grafana | CPU/内存/磁盘/网络 | CPU > 80% / 内存 > 85% |
| 应用性能 | SkyWalking APM | 调用链追踪、慢查询 | 响应时间 > 2s |
| 业务指标 | 自定义 Metrics | 订单量/支付成功率/AI 耗时 | 支付成功率 < 95% |
| 安全告警 | OWASP ZAP + 日志分析 | 异常访问/暴力破解 | 登录失败 > 10次/分钟 |
