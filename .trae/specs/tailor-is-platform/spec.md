# Tailor IS（裁智云）服装全产业平台项目规格

## 为什么

服装行业存在版型非标、版权混乱、交易无标准、结算不规范、协作低效、商户运营零散、产业链路割裂七大核心痛点。本项目旨在打造行业首个「AI智能制版引擎 + 区块链版权确权风控 + JAVA企业级多商户商城交易体系 + 产业协同工作台 + 内容社区生态」五位一体的垂直产业数字化平台，彻底解决行业痛点，实现从"设计创作→版权存证→多商户入驻开店→标准化商品上架→营销转化→担保交易→定制协作交付→售后工单履约→数据复盘迭代"的全链路商业闭环。

## 变更内容

- **ADDED**: 全新Tailor IS服装全产业平台，基于Java SpringBoot + Vue3 + UniApp技术栈
- **ADDED**: AI智能制版引擎模块
- **ADDED**: 区块链版权存证与风控模块
- **ADDED**: 多商户商城交易体系（平台总后台、商家后台、PC商城端、移动端H5/小程序）
- **ADDED**: 纸样定制可视化协作模块
- **ADDED**: 站内私信合规沟通模块
- **ADDED**: 种草社区内容生态模块
- **ADDED**: 知识学堂教学模块
- **ADDED**: 供应链供需对接模块
- **ADDED**: 数据服务与智能分析模块
- **ADDED**: 完整的代码质量管控、系统安全保障、性能优化、多平台兼容性、用户体验提升体系
- **ADDED**: 自动化质量保障机制（CI/CD、自动化测试、持续集成/部署）

## 影响范围

- **受影响系统**: 全新独立项目，参考Tailoris_backup但不直接复制
- **影响代码**: 全新代码结构，独立知识产权
- **技术依赖**: Java 17+, SpringBoot 3.x, Vue3, MySQL 8.0+, Redis 7.x+, RabbitMQ 3.12+

## ADDED 需求规格

### 需求1: 项目产品架构设计

系统 SHALL 提供完整的多端架构体系，包括：

#### 技术栈选型
- 后端：Java 17 + SpringBoot 3.2 + SpringCloud 2023 + MyBatis-Plus 3.5 + MySQL 8.0 + Redis 7.x + RabbitMQ 3.12
- 前端PC端：Vue3 + Vite + Element Plus + TypeScript
- 移动端：UniApp（H5/小程序跨端）
- 运维：Docker + Nginx + CDN + HTTPS
- 工具：Maven + Git + SonarQube + JMeter

#### 系统架构分层
- 表现层：PC商城端、商家后台、平台总后台、移动端H5/小程序
- 网关层：SpringCloud Gateway统一路由、鉴权、限流
- 业务层：商品服务、订单服务、用户服务、支付服务、营销服务、版权服务、社区服务
- 数据层：MySQL主从、Redis缓存集群、对象存储OSS
- 基础设施层：Docker容器化、Nginx负载均衡、CDN加速

#### 模块交互关系
- 网关 → 业务微服务（通过Nacos服务发现）
- 业务微服务 → 数据层（MySQL + Redis）
- 业务微服务 → 消息队列（RabbitMQ异步解耦）
- 外部系统 → AI服务中台（算法接口调用）

### 需求2: 多商户商城交易体系

系统 SHALL 提供完整的多商户商城功能：

#### 场景1: 商户入驻与分层
- WHEN 商户提交入驻申请
- THEN 系统记录资质信息，平台审核通过后分配商户类型（个人版师店/工作室店/品牌企业店/供应链商户店）
- THEN 商户获得独立后台、独立结算、独立风控权限

#### 场景2: 商品管理与上架
- WHEN 商家上架商品
- THEN 系统根据商品类型（数字纸样/定制服务/实物商品）应用差异化参数模板
- THEN 商品需经AI相似度检测与版权风控审核

#### 场景3: 订单交易流程
- WHEN 用户提交订单
- THEN 系统自动匹配优惠券，计算最终价格
- THEN 资金进入平台担保账户
- THEN 订单状态流转：待付款 → 待发货/待交付 → 已完成/已退款

#### 场景4: 售后工单处理
- WHEN 用户发起售后
- THEN 系统根据售后类型（仅退款/退货退款/定制返工）创建对应工单
- THEN 商家审核处理，系统记录全流程日志

#### 场景5: 资金结算分账
- WHEN 订单完成
- THEN 系统自动扣除平台技术服务费
- THEN 剩余资金进入商户可结算余额
- THEN 商户可申请提现，系统记录账单明细

### 需求3: 区块链版权存证

系统 SHALL 提供全链路版权保护：

#### 场景: 原创内容存证
- WHEN 商户上传原创纸样或设计作品
- THEN 系统自动抓取创作时间、作者信息、文件哈希值
- THEN 上链生成区块链存证证书
- THEN 支持72小时快速仲裁与侵权追溯

### 需求4: 代码质量管控

系统 SHALL 建立严格的代码质量保障体系：

#### 编码规范
- 命名规范：驼峰命名、语义化、统一前缀后缀
- 代码格式：统一缩进、行宽限制、导入顺序
- 注释要求：方法级JavaDoc、关键逻辑行注释
- 文件组织：按业务模块分层（controller/service/mapper/entity/dto/vo）

#### 代码审查流程
- 审查人员：技术负责人 + 模块Owner
- 审查标准：功能完整性、代码可读性、性能影响、安全隐患
- 审查频率：每次PR合并前强制审查

#### 自动化检测
- SonarQube集成：代码异味、漏洞、重复率检测
- 质量门禁：代码覆盖率≥80%，阻断问题0个，严重问题≤5个

### 需求5: 系统安全保障

系统 SHALL 实施全方位安全防护：

#### 身份认证与授权
- JWT Token认证，支持双Token刷新机制
- RBAC角色权限控制，精确到按钮级别
- 接口签名验证，防篡改防重放

#### 数据加密
- 传输加密：全站HTTPS，TLS 1.3
- 存储加密：敏感数据AES-256加密（密码、身份证、银行卡）
- 敏感信息脱敏：手机号中间四位、身份证部分隐藏

#### 漏洞防护
- SQL注入：MyBatis参数化查询，禁用动态拼接
- XSS防护：输入输出双重过滤，CSP策略
- CSRF防护：Token验证，SameSite Cookie
- 接口限流：Gateway层令牌桶算法，单IP 100次/分钟

### 需求6: 性能优化策略

系统 SHALL 提供高性能响应能力：

#### 数据库优化
- 索引设计：高频查询字段建立复合索引
- SQL优化：避免全表扫描，分页查询limit限制
- 查询缓存：热点数据Redis缓存，TTL 30分钟

#### 前端加载优化
- 资源压缩：Gzip/Brotli压缩，压缩率≥70%
- 懒加载：图片、路由组件按需加载
- CDN加速：静态资源CDN分发，缓存命中率达95%

#### 服务器响应优化
- 接口性能：核心接口响应时间≤200ms，P99≤500ms
- 缓存策略：多级缓存（本地Caffeine + Redis集群）
- 负载均衡：Nginx轮询/加权轮询，支持动态扩容
