# Tailor IS（裁智云）项目站内私信功能开发文档

## 一、功能概述

站内私信功能是Tailor IS项目的核心功能之一，旨在提供用户间安全、高效、实时的消息通信能力。该功能支持文本、图片、文件和站内链接的发送与接收，并具备完善的内容安全过滤机制和实时通信能力。

## 二、技术架构

### 2.1 技术栈

- **前端**：Vue 3 + TypeScript + Element Plus + Tailwind CSS
- **后端**：Spring Boot 3.x + MyBatis Plus + WebSocket + Redis
- **数据库**：MySQL 8.0
- **文件存储**：MinIO
- **安全过滤**：AC自动机算法 + 正则表达式

### 2.2 系统架构

## 三、核心功能实现

### 3.1 后端实现

#### 3.1.1 数据库设计

**`private_chat_session`表**：存储用户对话会话
- `id`：会话ID
- `user_id_1`：用户1 ID
- `user_id_2`：用户2 ID
- `last_message`：最后一条消息内容
- `last_message_time`：最后一条消息时间
- `unread_count_1`：用户1未读消息数
- `unread_count_2`：用户2未读消息数
- `status`：会话状态
- `create_time`：创建时间
- `update_time`：更新时间
- `is_deleted`：是否删除

**`private_chat_message`表**：存储私信消息
- `id`：消息ID
- `session_id`：会话ID
- `sender_id`：发送者ID
- `receiver_id`：接收者ID
- `content`：消息内容
- `type`：消息类型（1-文本，2-图片，3-文件，4-站内链接）
- `status`：消息状态（0-未读，1-已读，2-已撤回，3-已删除）
- `send_time`：发送时间
- `read_time`：已读时间
- `recall_time`：撤回时间
- `audit_status`：审核状态
- `audit_time`：审核时间
- `audit_user_id`：审核用户ID
- `create_time`：创建时间
- `update_time`：更新时间
- `is_deleted`：是否删除

**`private_chat_attachment`表**：存储私信附件
- `id`：附件ID
- `message_id`：消息ID
- `file_name`：文件名
- `file_type`：文件类型
- `file_size`：文件大小
- `file_path`：文件路径
- `file_url`：文件URL
- `thumbnail_url`：缩略图URL
- `upload_time`：上传时间
- `create_time`：创建时间
- `update_time`：更新时间
- `is_deleted`：是否删除

**`chat_message_audit`表**：存储消息审核记录
- `id`：审核ID
- `message_id`：消息ID
- `session_id`：会话ID
- `sender_id`：发送者ID
- `receiver_id`：接收者ID
- `message_content`：消息内容
- `violation_type`：违规类型
- `violation_detail`：违规详情
- `audit_status`：审核状态
- `audit_result`：审核结果
- `audit_user_id`：审核用户ID
- `audit_time`：审核时间
- `create_time`：创建时间
- `update_time`：更新时间
- `is_deleted`：是否删除

#### 3.1.2 REST API设计

| 接口路径 | 方法 | 功能描述 | 参数 | 返回值 |
|---------|------|----------|------|--------|
| `/api/private-chat/sessions` | POST | 创建或获取会话 | userId, targetUserId | PrivateChatSession |
| `/api/private-chat/sessions` | GET | 获取用户会话列表 | userId | List<PrivateChatSession> |
| `/api/private-chat/sessions/{sessionId}` | GET | 获取会话详情 | sessionId, userId | PrivateChatSession |
| `/api/private-chat/sessions/{sessionId}` | DELETE | 删除会话 | sessionId, userId | boolean |
| `/api/private-chat/messages` | POST | 发送消息 | sessionId, senderId, receiverId, content, type | PrivateChatMessage |
| `/api/private-chat/sessions/{sessionId}/messages` | GET | 获取会话消息历史 | sessionId, userId, offset, limit | List<PrivateChatMessage> |
| `/api/private-chat/messages/{messageId}/read` | PUT | 标记消息为已读 | messageId, userId | boolean |
| `/api/private-chat/sessions/{sessionId}/read` | PUT | 标记会话所有消息为已读 | sessionId, userId | boolean |
| `/api/private-chat/messages/{messageId}/recall` | PUT | 撤回消息 | messageId, userId | boolean |
| `/api/private-chat/messages/{messageId}` | DELETE | 删除消息 | messageId, userId | boolean |
| `/api/private-chat/unread-count` | GET | 获取未读消息数 | userId | integer |
| `/api/private-chat/upload` | POST | 上传附件 | file, messageId, userId | object |
| `/api/private-chat/messages/{messageId}/attachments` | GET | 获取消息附件 | messageId | List<object> |
| `/api/private-chat/sessions/{sessionId}/attachments` | GET | 获取会话附件 | sessionId | List<object> |
| `/api/private-chat/attachments/{attachmentId}` | DELETE | 删除附件 | attachmentId, userId | boolean |

#### 3.1.3 WebSocket实现

**连接管理**：
- 建立WebSocket连接（带Token认证）
- 实时消息推送（新消息、消息已读、消息撤回等事件）
- 连接状态管理（在线、离线、重连）
- 消息确认机制（确保消息送达）

**技术实现**：
- Spring WebSocket + STOMP协议
- Redis存储在线状态
- 消息持久化（保证离线消息不丢失）
- 心跳机制（维持连接）

#### 3.1.4 内容安全过滤

**过滤规则**：
1. **联系方式检测**（使用正则表达式）：
   - 手机号（中国大陆+国际格式）
   - 邮箱地址
   - 微信号/QQ号模式
   - 社交账号特征
2. **敏感词过滤**：
   - AC自动机算法匹配
   - 支持敏感词库动态更新
   - 支持模糊匹配（拼音、形近字）
3. **链接检测**：
   - 只允许站内链接（域名白名单）
   - 外部链接自动拦截

**功能要求**：
- 发送前实时检测
- 违规内容自动拦截并返回具体原因
- 可疑内容转入人工审核队列
- 支持学习模式（收集新敏感词）

## 七、部署指南

### 7.1 环境要求

- JDK 11+
- Maven 3.8+
- MySQL 8.0+
- Redis 7.0+
- MinIO
- Node.js 18+

### 7.2 部署步骤

1. **数据库初始化**：
   - 执行`private_chat_tables.sql`脚本创建数据库表结构
   - 初始化敏感词库

2. **后端部署**：
   ```bash
   cd tailoris-parent
   mvn clean package -DskipTests
   cd tailoris-message
   java -jar target/tailoris-message-1.0.0.jar
   ```

3. **前端部署**：
   ```bash
   cd tailoris-frontend
   npm install
   npm run build
   # 将dist目录部署到Nginx或其他静态文件服务器
   ```

4. **配置说明**：
   - 后端配置：修改`application.yml`中的数据库连接、Redis连接、MinIO连接等配置
   - 前端配置：修改API接口地址配置

## 八、使用指南

### 8.1 用户使用指南

#### 8.1.1 发送私信

1. **进入私信页面**：
   - 登录系统后，点击顶部导航栏的"私信"图标

2. **选择对话**：
   - 在左侧会话列表中选择一个现有对话
   - 或点击"发起私信"按钮创建新对话

3. **发送消息**：
   - 在消息输入框中输入文本
   - 点击"发送"按钮发送消息

4. **发送图片/文件**：
   - 点击输入框右侧的"图片"或"文件"图标
   - 选择要上传的文件
   - 点击"发送"按钮发送

5. **查看消息状态**：
   - 消息右侧显示"未读"或"已读"状态
   - 会话列表显示未读消息数

#### 8.1.2 管理私信

1. **撤回消息**：
   - 右键点击已发送的消息
   - 选择"撤回消息"（2分钟内可撤回）

2. **删除消息**：
   - 右键点击消息
   - 选择"删除消息"（仅对己方不可见）

3. **标记已读**：
   - 打开对话自动标记为已读
   - 或点击会话列表中的对话标记为已读

### 8.2 管理员使用指南

#### 8.2.1 消息监控

1. **进入消息监控页面**：
   - 登录管理后台
   - 点击"内容管理" → "消息监控"

2. **查看消息**：
   - 浏览消息列表
   - 使用搜索条件筛选消息
   - 点击"详情"查看消息详情

3. **拦截消息**：
   - 点击"拦截"按钮拦截违规消息

#### 8.2.2 消息审核

1. **进入审核队列**：
   - 登录管理后台
   - 点击"内容管理" → "消息审核队列"

2. **审核消息**：
   - 浏览待审核消息列表
   - 点击"详情"查看消息详情和检测结果
   - 点击"通过"或"拒绝"按钮完成审核

3. **批量操作**：
   - 选择多条消息
   - 点击"批量通过"或"批量拒绝"按钮

#### 8.2.3 敏感词管理

1. **进入敏感词管理**：
   - 登录管理后台
   - 点击"商品管理" → "敏感词管理"

2. **管理敏感词**：
   - 点击"新增敏感词"添加新的敏感词
   - 点击"编辑"修改现有敏感词
   - 点击"删除"删除敏感词
   - 点击"启用"/"禁用"切换敏感词状态

## 九、问题与解决方案

### 9.1 已知问题

1. **WebSocket重连机制**：在网络不稳定情况下，重连可能会有短暂延迟
2. **文件上传速度**：大文件上传速度受网络带宽影响
3. **内容过滤误判**：部分正常内容可能被误判为违规内容

### 9.2 解决方案

1. **WebSocket重连优化**：
   - 优化重连策略，增加重连间隔
   - 实现指数退避重连算法

2. **文件上传优化**：
   - 实现文件分片上传
   - 增加上传进度显示
   - 优化文件存储路径

3. **内容过滤优化**：
   - 定期更新敏感词库
   - 优化过滤算法，减少误判
   - 增加人工审核机制
