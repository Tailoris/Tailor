# Tailor IS（裁智云）项目部署核查验证执行办法

### 一、核查验证范围与基准

本模块围绕 Tailor IS（裁智云）项目部署至 1Panel 生产环境的全流程开展核查验证，以《Tailor IS（裁智云）项目开发及部署专项核查工作指令》为核心基准，覆盖**部署架构合规性、环境配置准确性、服务运行稳定性、运维体系完整性、安全部署达标性、性能适配性**六大维度，确保部署结果完全匹配生产准入标准与服装产业平台业务诉求。

### 二、部署架构核查验证

#### 1\. 物理 / 逻辑部署架构一致性验证

|核查项|验证标准|验证方法|结果判定|问题记录|
|---|---|---|---|---|
|逻辑架构落地|与《系统架构文档》中逻辑架构图一致，核心业务 / 支撑服务 / 公共组件模块划分清晰，调用关系、数据流向无偏差|对照架构图梳理实际部署模块依赖关系，核验模块间接口调用链路|符合 / 不符合 / 部分符合|（示例：公共组件模块中缓存服务未按架构图部署至独立节点，存在与核心业务模块混部）|
|物理部署拓扑|1Panel 环境中服务器节点、容器 / 进程分布、网络拓扑与物理部署架构图一致，无节点缺失 / 冗余|登录 1Panel 控制台核查节点数量、角色（主 / 从 / 负载节点）、网络网段划分|符合 / 不符合 / 部分符合|\-|
|模块拆分合规性|无跨模块耦合部署、核心交易模块（纸样定制、订单交易）独立部署且具备容灾能力|核查 1Panel 中容器编排配置，验证核心模块资源隔离性|符合 / 不符合 / 部分符合|\-|

#### 2\. 技术栈部署匹配性验证

|技术栈类型|验证内容|验证工具 / 方式|结果|优化建议|
|---|---|---|---|---|
|后端技术栈|后端语言版本、框架版本、中间件（Redis / 消息队列）版本与《技术栈选型核验报告》一致，无版本兼容问题|执行 `java -version`/`node -v` 等命令，核查中间件配置文件版本|匹配 / 不匹配|（示例：Redis 版本为 5\.0，建议升级至 6\.2 以适配缓存穿透防护策略）|
|数据库|MySQL/PostgreSQL 版本、读写分离配置、分库分表规则落地，与数据库设计文档一致|登录数据库执行 `show variables like '%version%'`，核查读写分离路由规则|匹配 / 不匹配|\-|
|前端技术栈|前端框架版本、构建工具（Webpack/Vite）配置、依赖包版本无安全漏洞|执行 `npm audit` 扫描依赖漏洞，核查打包配置文件|匹配 / 无漏洞 / 存在漏洞|\-|

### 三、1Panel 环境配置核查验证

#### 1\. 基础配置核验（核心必验项）

|配置类别|细分核查项|验证标准|验证步骤|异常处理方式||
|---|---|---|---|---|---|
|环境变量|数据库连接串、密钥、第三方接口地址、环境标识（prod/test）|无硬编码、敏感信息加密存储、与生产环境配置一致|1\. 登录 1Panel 查看环境变量配置；2\. 核验配置值与《部署手册》一致性；3\. 验证变量生效状态|发现硬编码立即整改，敏感信息重新加密配置||
|目录权限|应用运行目录、日志目录、数据挂载目录|权限为 755（目录）/644（文件），运行用户为非 root 账户|执行 `ls -l` 核查目录权限，\`ps \-ef|grep 应用进程 \` 核查运行用户|调整权限命令：`chmod 755 /data/tailor-is`，切换运行用户|
|端口映射|应用端口、数据库端口、Nginx 反向代理端口|无端口冲突，对外暴露端口仅开放必要端口（80/443 / 业务端口），防火墙策略放行|1\. 执行 `netstat -tulpn` 核查端口占用；2\. 核查 1Panel 端口映射规则；3\. 测试端口连通性|冲突端口重新分配，关闭非必要开放端口||
|域名与证书|域名解析指向正确、HTTPS 证书有效（有效期＞30 天）、全站 HTTPS 配置生效|1\. ping 域名核查解析；2\. 访问域名查看证书信息；3\. 测试 HTTP 自动跳转 HTTPS|证书过期立即替换，重新配置 Nginx HTTPS 规则|||
|服务启停配置|应用服务、中间件服务开机自启，启停脚本无语法错误|1\. 核查 1Panel 服务自启配置；2\. 执行启停脚本测试；3\. 验证服务异常退出后自动重启|修复启停脚本错误，重新配置自启规则|||

#### 2\. 运维配置核验

|运维配置项|验证标准|验证方法|验收指标|
|---|---|---|---|

|定时任务|数据备份、缓存清理、统计报表生成等定时任务配置正确，执行日志可追溯|核查 1Panel 定时任务列表，查看历史执行日志|定时任务执行成功率 100%，无重复 / 遗漏执行|
|数据挂载|数据库数据、静态资源（3D 模型 / 视频）挂载至独立存储，支持扩容|核查 1Panel 存储挂载配置，测试存储扩容操作|挂载路径无硬编码，扩容后服务无需重启|
|备份策略|数据库全量备份（每日）\+ 增量备份（每小时），备份文件存储至异地，保留 30 天|核查备份脚本、备份文件列表、异地同步配置|备份文件可正常恢复，恢复时长≤30 分钟|

### 四、服务运行状态核查验证

#### 1\. 服务可用性验证

|服务类型|验证指标|验证方法|验收标准|
|---|---|---|---|
|核心业务服务|服务常驻运行，无异常退出，重启成功率 100%|1\. 核查 1Panel 服务状态（运行 / 停止 / 异常）；2\. 执行重启操作测试；3\. 监控服务运行时长|连续运行 72 小时无异常退出，重启耗时≤30 秒|
|集群 / 负载均衡|负载均衡策略（轮询 / 加权）生效，节点故障自动切换|1\. 模拟单个节点宕机，验证请求自动路由至正常节点；2\. 核查负载均衡日志|切换时长≤5 秒，无请求丢失 / 错误|
|第三方接口服务|与面辅料供应链、支付、版权存证等第三方接口连通性|调用接口测试用例，验证接口返回状态|接口调用成功率≥99\.9%，超时时间≤1 秒|

#### 2\. 并发与性能适配验证

|验证维度|测试场景|验证工具|验收标准（匹配生产准入指标）|

### 五、部署安全核查验证

#### 1\. 部署层安全防护

|安全核查项|验证标准|验证方法|整改要求|
|---|---|---|---|
|容器 / 进程安全|应用进程以最小权限运行，容器无特权模式，禁止挂载宿主机敏感目录|核查 1Panel 容器运行配置，执行 `ps -ef` 核查进程权限|关闭特权模式，移除不必要的宿主机目录挂载|
|接口安全部署|接口鉴权、签名校验、限流配置在 1Panel 网关层生效|1\. 测试无签名接口调用是否被拦截；2\. 核查网关限流规则（QPS 阈值）；3\. 模拟高频调用验证限流生效|鉴权失败拦截率 100%，限流阈值符合防刷策略（核心接口 QPS≥500）|
|依赖包安全|部署环境中第三方依赖无 OWASP 高危漏洞|执行 `maven dependency-check`/`npm audit` 扫描，对比漏洞库|高危漏洞依赖包立即升级，中低风险按需整改|

#### 2\. 数据安全部署验证

|数据安全项|验证内容|验证步骤|验收结果|
|---|---|---|---|
|传输加密|应用与数据库、应用与前端、第三方接口通信均为 HTTPS/WSS|1\. 抓包验证通信加密；2\. 核查数据库 SSL 连接配置|全链路加密生效，无明文传输|
|存储加密|敏感字段（手机号 / 身份证 / 交易数据）加密存储，与《安全测试报告》要求一致|登录数据库查看敏感字段存储格式，验证解密逻辑|加密算法符合要求（AES\-256），解密正常|

### 六、部署问题分级与整改闭环

#### 1\. 部署问题分级标准（对齐指令要求）

|等级|定义|整改时限|示例|
|---|---|---|---|
|Critical（P0）|部署配置错误导致核心服务宕机、数据挂载失败、域名无法访问、高危安全漏洞|立即停工修复（0\-2 小时）|数据库连接串配置错误导致订单服务无法启动；HTTPS 证书失效导致全站无法访问|
|High（P1）|非核心服务异常、端口映射错误（非核心端口）、日志采集失效、备份策略未落地|24 小时内修复|静态资源 CDN 配置错误导致加载缓慢；定时备份脚本执行失败|
|Medium（P2）|目录权限非最优配置、运维文档与实际部署不一致、非核心依赖版本略低|3 个工作日内修复|日志目录权限为 777（需调整为 755）；部署手册中未记录新增的定时任务|
|Low（P3）|部署注释不规范、1Panel 控制台展示名称不统一、非必要环境变量冗余|迭代周期内整改|容器名称存在错别字；冗余的测试环境变量未删除|

#### 2\. 部署问题闭环流程

1. 问题录入：所有部署问题按统一模板录入项目管理工具（Jira / 禅道），包含「问题 ID、等级、类型、描述、发现位置、复现步骤、责任人」；

2. 整改实施：责任人按优先级（P0\>P1\>P2\>P3）整改，核心业务模块问题优先；

3. 验证验收：整改完成后，部署负责人执行专项验证，确认问题修复无次生影响；

4. 状态更新：验证通过后更新问题状态为「已修复 / 已验收」，未通过则退回重改并记录根因；

5. 复盘归档：所有部署问题整改完成后，输出《部署问题整改验收清单》，纳入项目交付资料。

### 七、部署验证输出物

1. 《1Panel 部署配置核查记录表》（含所有配置项验证结果、异常项及整改记录）；

2. 《部署架构匹配性验证报告》（含架构落地偏差、技术栈适配性分析）；

3. 《服务运行稳定性测试报告》（含 72 小时运行监控数据、并发压测结果）；

4. 《部署安全核查清单》（含安全防护落地情况、漏洞整改记录）；

5. 《部署问题整改验收清单》（含所有分级问题的整改状态、验收结果）；

6. 《1Panel 生产环境部署验收报告》（综合验证结论、是否达标、上线建议）。

### 八、部署验收判定规则

只有满足以下全部条件，判定为部署验证通过：

1. 部署架构、环境配置、服务运行、安全防护四类核心核查项通过率 100%；

2. P0/P1 级部署问题全部清零，P2 级问题整改完成率 100%，P3 级问题明确整改计划；

3. 性能适配验证中，并发访问、接口响应、资源占用等指标达到生产准入标准；

4. 所有部署验证输出物完整、准确，与实际部署状态一致；

5. 1Panel 生产环境中系统连续稳定运行 72 小时无异常，备份 / 监控 / 运维体系可正常使用。
