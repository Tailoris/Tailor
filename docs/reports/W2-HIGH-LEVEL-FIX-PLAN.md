## 2. 关键技术决策

### 2.1 AES加密策略

**B-H01 + B-H25 + B-H26 统一方案**:
- 算法：AES-256-GCM（认证加密，防篡改）
- 密钥管理：配置中心（Nacos）+ 本地缓存
- 密钥轮换：每90天自动轮换
- IV生成：每次加密使用随机IV
- 兼容性：保留CBC模式用于老数据

### 2.2 XSS + CSRF 防护

**B-H19 + B-H23 统一方案**:
- XSS：基于OWASP Java Encoder + 自定义白名单
- CSRF：Sa-Token CSRF Token机制
- Token生成：每会话一个CSRF Token
- 验证：POST/PUT/DELETE强制验证

### 2.3 N+1查询修复

**B-H18 + TD-04 统一方案**:
- 工具：MyBatis-Plus的JOIN查询
- 缓存：热点数据预加载
- 监控：慢查询日志+SQL分析
- 目标：单接口查询次数<3

### 2.4 Seata分布式事务

**TD-10 实施方案**:
- 模式：AT模式（自动补偿）
- TC：独立部署Seata Server
- 配置：自动代理DataSource
- 范围：订单-支付-库存-分账
