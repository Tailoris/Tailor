## 1. 风险评估矩阵

### 1.1 风险等级定义

| 等级 | 名称 | 影响 | 颜色 |
|------|------|------|:----:|
| 严重 | Critical | 系统无法运行 | 🔴 |
| 高 | High | 核心功能受损 | 🟠 |
| 中 | Medium | 部分功能受损 | 🟡 |
| 低 | Low | 轻微影响 | 🟢 |

### 1.2 风险等级计算

**风险等级 = 可能性 × 影响**

- 可能性: 1（低）~ 5（高）
- 影响: 1（低）~ 5（高）
- 风险值: 1~25
- 等级: 1-5=低, 6-10=中, 11-15=高, 16-25=严重

## 3. 风险应对预案

### 3.1 高风险预案（橙色 + 红色）

#### 预案 A: 数据库迁移失败（R11）

**触发**: 数据库导入过程中断或报错

**RTO**: 30 分钟
**RPO**: 0（迁移前有完整备份）

**步骤**:
1. 立即停止迁移操作
2. 检查错误日志，定位失败原因
3. 评估影响范围：受影响表/数据量
4. **情况 1（部分失败）**: 修复后重新执行（重试 3 次）
5. **情况 2（完全失败）**:
   - 不导入新数据，保留源数据库
   - 启动应急流程
   - 通知所有相关方
6. **情况 3（数据损坏）**:
   - 立即恢复备份
   - 启动根因分析
   - 准备重新部署

**负责人**: DBA
**联系**: 138-0000-0003

#### 预案 B: 灰度期间严重 Bug（R21）

**触发**: 灰度期间错误率 >1% 或 P99 响应时间 >5s

**RTO**: 15 分钟

**步骤**:
1. SRE 立即响应（5 分钟内）
2. 评估影响范围：受影响服务/用户量
3. **决策树**:
   - 单服务 Bug → 回滚单个服务
   - 多服务 Bug → 回滚整个版本
   - 数据问题 → 数据库回滚
4. 执行回滚：`bash 1panel-rollback.sh all`
5. 验证回滚成功
6. 通知相关方
7. 启动 Post-Mortem

**负责人**: SRE
**联系**: 138-0000-0007

#### 预案 C: 流量激增导致雪崩（R22）

**触发**: QPS 突增 5x 或服务错误率 >5%

**RTO**: 5 分钟

**步骤**:
1. 立即启用限流：
   - Nginx 限流（已配置 50r/s 通用）
   - Sentinel 熔断降级
2. 紧急扩容：
   - 增加服务节点
   - 增加数据库连接
3. 静态化关键页面：
   - 首页
   - 列表页
   - 详情页
4. 关闭非核心功能：
   - 推荐系统
   - 数据统计
5. 启用 CDN 缓存
6. 通知用户（如必要）

**负责人**: SRE
**联系**: 138-0000-0007

#### 预案 D: 数据库连接池耗尽（R24）

**触发**: HikariCP active/max > 90%

**RTO**: 10 分钟

**步骤**:
1. 立即启用应用层限流
2. 查找慢查询并 kill
3. 优化长事务
4. 重启应用释放连接
5. 增加连接池大小（如可）
6. 排查根本原因

**负责人**: DBA + 后端

#### 预案 E: 磁盘空间不足（R10）

**触发**: 磁盘使用率 >85%

**RTO**: 30 分钟

**步骤**:
1. 立即清理：
   - 旧日志（保留 7 天）
   - 旧备份（保留 30 天）
   - 临时文件
2. 扩容磁盘（如云盘支持）
3. 调整日志级别（DEBUG → INFO）
4. 启用日志压缩

**负责人**: DevOps

### 3.2 中等风险预案

#### 预案 F: 服务启动失败（R14）

**步骤**:
1. 查看启动日志：`tail -200 /opt/tailor-is/logs/{service}.log`
2. 常见原因排查：
   - 端口冲突：`netstat -tlnp | grep <port>`
   - 配置错误：检查 Nacos 配置
   - 依赖服务未启动：先启动 MySQL/Redis/Nacos
   - 资源不足：检查内存/磁盘
3. 修复后重新启动
4. 仍失败则回滚

#### 预案 G: Nginx 配置错误（R19）

**步骤**:
1. 测试配置：`sudo nginx -t`
2. 查看错误日志：`/var/log/nginx/error.log`
3. 恢复备份配置
4. 重新加载：`sudo systemctl reload nginx`

#### 预案 H: HTTPS 证书过期（R20）

**步骤**:
1. certbot 自动续期：`sudo certbot renew`
2. 重启 Nginx：`sudo systemctl restart nginx`
3. 验证：`curl -I https://tailor-is.com`
